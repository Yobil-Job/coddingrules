Code Writing Rules

Best Practices for Website and Backend Developers

1. Naming Conventions

camelCase: Used for variables and functions. Example: let userName = "JohnDoe";

PascalCase: Used for class names. Example: class UserProfile {}

snake_case: Used in Python and databases. Example: user_name = "JohnDoe"

kebab-case: Used in HTML/CSS. Example: <div class="user-profile"></div>

UPPER_CASE: Used for constants. Example: const MAX_USERS = 100;

2. Code Formatting and Structure

Indentation: Use 2 or 4 spaces consistently.

Line Length: Limit lines to 80-120 characters.

Braces and Brackets: Always use braces for blocks.

Spacing: Use spaces around operators and after commas.

3. Comments and Documentation

Inline Comments: Explain non-obvious code.

Function and Class Comments: Use JSDoc or similar tools.

File Headers: Include a brief description at the top of each file.

4. Error Handling

Try-Catch Blocks: Wrap error-prone code in try-catch.

Validate Inputs: Always validate user inputs.

Log Errors: Log errors with sufficient context.

5. Performance Optimization

Minimize DOM Manipulations: Reduce DOM updates.

Optimize Database Queries: Use indexing and caching.

Use Asynchronous Operations: Use async/await or promises.

6. Version Control and Collaboration

Commit Messages: Write clear and concise messages.

Branching Strategy: Use Git Flow or GitHub Flow.

Code Reviews: Conduct regular code reviews.

7. Security Best Practices

Sanitize Inputs: Prevent SQL injection and XSS.

Use HTTPS: Encrypt data transmissions.

Authentication and Authorization: Use OAuth or JWT.
